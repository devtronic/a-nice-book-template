# Preface {.unnumbered}

This book is written with the very template it describes.

Every code listing, table, callout box, and diagram you encounter in these pages was produced by the system documented here. The PDF you are reading was generated by running `make pdf` from a plain-text source tree that you can clone, fork, and make your own.

## Why This Exists

Technical authors have long faced an unpleasant choice: use a word processor and fight its limitations, or invest weeks configuring a toolchain before writing a single word. The goal of this template is to eliminate that choice. It gives you a production-quality typesetting pipeline on day one, so you can focus on what you actually want to do — write.

The template is built on three time-tested tools:

- **Pandoc** converts Markdown to PDF, EPUB, and HTML from a single source.
- **XeLaTeX** handles the PDF rendering with full OpenType font support.
- **Make** ties everything together with a handful of memorable targets.

Everything is plain text. Your chapters are Markdown files. Your bibliography is BibTeX. Your diagrams are Mermaid code. All of it lives in a Git repository and diffs cleanly.

## How to Read This Book

Chapter 1 covers installation. If you have already run `make pdf` successfully, skim it for context and move on.

Chapters 2 through 6 cover the authoring features — writing prose, formatting code, building tables, inserting diagrams, and using callout boxes. Each chapter is both documentation and a live example: the features are demonstrated by using them.

Chapter 7 covers the build system: the Makefile targets, excerpt generation, and output formats.

Chapter 8 covers customization — changing fonts, colors, and code themes to match your book's identity.

## Conventions

When this book shows terminal commands, they appear in a code block with the `$` prompt:

```bash
$ make pdf
```

Inline file paths and configuration keys appear in `monospace`. Names of Makefile targets appear as `make target`.

Callout boxes highlight information that deserves special attention. You will see three varieties throughout:

```{=latex}
\begin{tipbox}
A tip offers a shortcut or a better approach you might not have considered.
\end{tipbox}
```

```{=latex}
\begin{warningbox}
A warning flags something that commonly goes wrong and how to avoid it.
\end{warningbox}
```

```{=latex}
\begin{dangerbox}
A danger box marks an action that can cause data loss or a broken build.
\end{dangerbox}
```
